複数のプロジェクトをVS Codeで開くとき、ウィンドウが増えすぎて作業スペースが散らかってしまうことはありませんか？

macOSのVS Codeには、複数のプロジェクトを1つのウィンドウ内でタブ管理できる便利な機能があります。しかし、この機能を有効にするには**VS Code側の設定**と**macOS側の設定**の両方が必要です。

ネット上の記事では片方の設定しか載っていないことが多く、「設定したのに動かない」と困った経験がある方も多いのではないでしょうか。

本記事では、VS Codeのネイティブタブ機能を確実に有効にする方法を、macOSのシステム設定まで含めて詳しく解説します。

## この機能でできること

- 複数のVS Codeプロジェクトを1つのウィンドウ内でタブとして管理
- Safariやブラウザのように、タブの切り替えで簡単にプロジェクト間を移動
- ウィンドウが散らからず、デスクトップがすっきり
- macOSのネイティブタブ機能を使用するため、他のmacアプリと同じ操作感


## 注意：この機能はmacOS限定です

`Window: Native Tabs`という名前の通り、この機能は**macOSでのみ利用可能**です。WindowsやLinuxでは使用できません。

## 設定手順

設定は2段階に分かれています。両方の設定を行わないと機能しないので注意してください。

### 1. VS Code側の設定

#### 方法1: GUIから設定（推奨）

1. VS Codeの設定画面を開く
   - macOS: `Cmd + ,`
   - メニューバー: `Code` → `Settings...` → `Settings`

2. 設定画面の検索ボックスで「native tabs」を検索


3. `Window: Native Tabs`にチェックを入れる


4. タイトルバーのスタイルも設定する
   - 「title bar」で検索
   - `Window: Title Bar Style`を`native`に変更


#### 方法2: settings.jsonで設定

`settings.json`に以下を追加します：

```json
{
  "window.nativeTabs": true,
  "window.titleBarStyle": "native"
}
```

`settings.json`を開く方法：
- `Cmd + Shift + P`でコマンドパレットを開く
- 「Preferences: Open User Settings (JSON)」を検索して実行

### 2. macOS側の設定

VS Codeの設定だけでは不十分です。**macOSのシステム設定でタブ表示を有効にする必要があります**。

この設定を忘れると、VS Code側で設定しても新規ウィンドウがタブとして開かれません。

1. システム設定を開く
2. 「デスクトップとDock」をクリック
3. 下にスクロールして「ウインドウ」セクションを探す
4. 「書類を開くときはタブで開く」を **「常に」** に設定


## 実際の使い方

設定が完了したら、以下の方法でタブを活用できます。

### 新規プロジェクトをタブで開く

1. 既にVS Codeでプロジェクトを開いている状態で、新しいプロジェクトを開く
2. `File` → `New Window`で新規ウィンドウを開く
3. タブとして開かれます

### 既存のウィンドウをタブに統合

複数のVS Codeウィンドウが既に開いている場合：

1. いずれかのウィンドウのメニューバーで`Window`をクリック
2. `Merge All Windows`（すべてのウインドウを統合）を選択
3. すべてのウィンドウが1つにまとまり、タブとして表示されます


### タブの操作

- **タブの切り替え**: `Cmd + Shift + [` / `Cmd + Shift + ]`
- **タブを閉じる**: タブ上の×ボタン、または`Cmd + W`
- **タブを新しいウィンドウに分離**: タブをデスクトップにドラッグ
- **タブの並び替え**: タブをドラッグ&ドロップ


## トラブルシューティング

### 設定したのにタブとして開かれない

以下を確認してください：

1. VS Code側の設定
   - `window.nativeTabs`が`true`になっているか
   - `window.titleBarStyle`が`native`になっているか

2. macOS側の設定
   - システム設定で「常に」が選択されているか
   - macOSのバージョンに合った場所で設定しているか

3. VS Codeの再起動
   - 設定変更後、VS Codeを完全に再起動してください
   - `Cmd + Q`で完全終了してから再度開く

### タイトルバーの見た目が変わった

`window.titleBarStyle`を`native`に変更すると、タイトルバーがmacOSネイティブのスタイルになります。これは正常な動作です。

元のカスタムタイトルバーに戻したい場合は、`window.titleBarStyle`を`custom`に変更してください。ただし、その場合はネイティブタブ機能も無効になります。

### 一部のウィンドウだけタブにならない

`File` → `New Window`で開いた空のウィンドウではなく、フォルダを開いたウィンドウ同士でないとタブとして統合されないことがあります。

## まとめ

VS Codeのネイティブタブ機能を有効にするには：

1. **VS Code側**: `window.nativeTabs`と`window.titleBarStyle`を設定
2. **macOS側**: システム設定で「書類を開くときはタブで開く」を「常に」に設定

両方の設定を行って初めて機能します。特にmacOS側の設定は忘れがちなので、動かない場合はまずシステム設定を確認してみてください。

この設定により、複数プロジェクトでの作業効率が大幅に向上します。ぜひ試してみてください。
